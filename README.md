## 快速开始

### 环境要求

- Windows 10/11
- Python 3.8+
- 网络连接（天气查询、NTP 校时需要）

### 1. 安装依赖

```powershell
cd desktop_pet
python -m pip install -r requirements.txt
```

如遇网络问题，使用清华镜像：

```powershell
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

### 2. 运行程序

```powershell
python main.py
```

程序启动后：
1. 系统托盘出现宠物图标 🐾
2. 宠物出现在桌面右下角（默认安静模式）
3. 右键托盘图标打开操作菜单

### 3. 首次配置

首次运行时会自动将 `config.yaml` 复制到 `data/config.yaml`（exe 同级目录）。

推荐先通过 **"管理提醒"** 面板配置你的打卡时间和提醒文案，然后保存即可。

### 宠物交互

| 操作 | 效果 |
|------|------|
| **拖拽宠物** | 按住左键拖动，可移动到屏幕任意位置 |
| **右键宠物** | 弹出快捷菜单（活跃/安静模式切换、退出） |
| **闹钟触发** | 根据动作类型播放动画或打开 URL，并显示可操作的提醒气泡 |
| **托盘隐藏时触发** | 动画提醒会临时显示宠物，处理完提醒队列后恢复隐藏 |
| **安静模式** | 宠物停在右下角，显示静态图片，不闲逛 |
| **活跃模式** | 宠物在屏幕中央区域左右移动，播放循环动画 |

### 小新的专注大作战

通过托盘菜单 → **“番茄钟” → “打开番茄钟”** 进入漫画风控制面板：

1. 可选择15、25、45、60分钟快捷时长，并填写本轮目标。
2. 点击“开始专注”后默认自动隐藏宠物，托盘菜单持续显示剩余时间。
3. 支持暂停、继续和提前结束；关闭控制面板不会停止计时。
4. 专注完成后宠物临时出现并播放庆祝动画，默认等待手动开始休息。
5. 每完成4轮进入15分钟长休息；轮数和专注分钟按日期保存。

默认节奏为专注25分钟、短休息5分钟、长休息15分钟。设置页可以开启自动开始
休息或自动开始下一轮。计时使用绝对结束时间，锁屏、休眠或重启后仍能正确恢复。

番茄钟记录保存在 `data/pomodoro.yaml`，采用原子写入和 `.bak` 备份。普通定时提醒
优先级高于番茄钟完成提示；冲突时先处理提醒，再展示番茄钟结果。

### 管理提醒任务

通过托盘菜单 → **"管理提醒"** 打开管理面板：

#### 新增提醒

1. 点击 **"新增提醒"** 按钮
2. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 提醒名称 | 任务的简短描述 | `下班打卡` |
| 触发时间 | 24小时制 HH:MM | `18:30` |
| 触发日期 | 每天、仅工作日、仅休息日或指定日期（一次性） | `仅休息日` |
| 具体日期 | 选择一次性提醒时设置年月日 | `2026-08-15` |
| 动作类型 | 触发时的行为，新增提醒默认播放动画 | `播放动画` / `打开URL` / `仅通知` |
| 目标URL | 动作类型为"打开URL"时填写 | `dingtalk://...` |
| 专属动画 | 触发时播放的动画 | `cheer` / `walk` |
| 提醒文案 | 气泡中显示的文本 | `下班啦！快去打卡~` |
| 播放提示音 | 是否播放音效 | ☑ |
| **专属音效** | 选择此提醒专用的 WAV 文件（留空则用默认提示音） | `小新-下班了.wav` |

3. 点击 **"确定"** 保存

#### 编辑/删除提醒

- 在列表中选中一行，点击 **"编辑"** 修改配置
- 点击 **"删除"** 移除该提醒
- 勾选/取消勾选启用列，快速开关提醒
- 一次性提醒触发后显示“已完成”并自动停用；超过触发时间24小时显示“已过期”
- 已完成或已过期的提醒需要先编辑为未来时间，才能重新启用

#### 刷新列表

修改保存后自动刷新，无需重启程序。

#### 贪睡与完成

提醒触发时，动画和交互气泡不会自动消失，只能通过以下按钮结束当前展示：

- **稍后10分钟哈~**：关闭气泡和动画并恢复安静模式，10 分钟后重新提醒
- **知道了~**：完成本次提醒，关闭气泡和动画并恢复安静模式

多条提醒按实际触发时间 FIFO 排队，一次只展示一条。待处理记录写入
`data/pending_reminders.yaml`，程序重启后继续处理；未展示记录最多保留 24 小时。

锁屏期间不会播放声音或展示动画，解锁后按队列恢复且每条只响一次。休眠唤醒后会扫描
遗漏时间窗，最多追溯 24 小时。一次性提醒即使程序当时未运行，也会在24小时内补发；
超过24小时后标记为已过期。

### 配置说明

配置文件位于 `data/config.yaml`（exe 同级目录），支持以下配置项：

#### 宠物基础配置

```yaml
pet:
  name: "小新"              # 宠物名称
  style: "shinchan"         # 风格标识
  default_animation: "cheer" # 默认动画
```

#### 提醒配置

```yaml
reminders:
  - name: "下班打卡"
    enabled: true             # 是否启用
    time: "18:30"            # 触发时间（24小时制）
    schedule_type: "daily"   # daily / workday / rest_day / once
    action_type: "play_animation" # play_animation / open_url / notify_only
    action_target: ""          # open_url 类型的目标URL
    animation: "cheer"        # 专属动画
    message: "下班啦！快去打卡~"     # 提醒文案
    sound: true               # 是否播放提示音
    sound_file: ""            # 专属音效文件路径（空=使用默认）

  - id: "one-time-example"   # 一次性提醒必须有唯一 id
    name: "一次性提醒"
    enabled: true
    date: "2026-08-15"       # once 专用，YYYY-MM-DD
    time: "09:00"
    schedule_type: "once"
    status: "pending"        # pending / completed / expired
    action_type: "play_animation"
    animation: "cheer"
    message: "一次性提醒时间到了~"
    sound: true
```

`weekdays_only` 旧字段仍可读取：`true` 等价于 `workday`，`false` 等价于 `daily`。
编辑旧提醒并保存后会自动转换为 `schedule_type`。

`rest_day` 表示普通周六、周日以及法定节假日，并排除调休上班日。程序内置2026年
中国大陆日历，`holidays` 手工配置优先级更高。2027年正式安排尚未发布，缺失年份会记录
告警并暂按周一至周五判断。

#### 网络时间校准

```yaml
time_sync:
  enabled: true
  ntp_server: "ntp.aliyun.com"   # NTP 服务器
  tolerance_seconds: 30          # 偏差告警阈值（秒）
```

#### 番茄钟配置

```yaml
pomodoro:
  focus_minutes: 25
  short_break_minutes: 5
  long_break_minutes: 15
  long_break_every: 4
  auto_start_break: false
  auto_start_focus: false
  hide_during_focus: true
```

#### 天气配置

```yaml
weather:
  enabled: true
  city: "北京"                    # 查询城市
  api_provider: "openmeteo"       # openmeteo(免Key) / qweather / openweathermap
  api_key: ""                     # 和风天气/OpenWeatherMap 需填入 Key
  update_interval_minutes: 60     # 同一城市缓存刷新间隔（分钟）
```

#### 提醒引擎配置

```yaml
engine:
  check_interval_sec: 1
  sleep_grace_period_sec: 60
  missed_reminder_retention_hours: 24 # 最大值固定为 24 小时
```

### 自定义音效

#### 支持的格式

- **WAV 格式**（推荐）— 使用 `winsound` 播放，零额外依赖，无窗口闪烁
- 文件放在 `assets/sounds/` 目录下

#### 设置默认提示音

将默认提示音文件命名为 `reminder.wav`，放在 `assets/sounds/` 目录。

#### 为特定提醒设置专属音效

1. 将 WAV 文件放入 `assets/sounds/` 目录
2. 在提醒管理面板中，选择该提醒的 **"专属音效"** 下拉框
3. 选择对应的 WAV 文件
4. 保存即可

> 💡 **小技巧**：可以用 Python 脚本裁剪现有 WAV 文件的前几秒作为提示音：
> ```python
> import wave, os
> with wave.open('source.wav', 'rb') as w:
>     params = w.getparams()
>     frames = w.readframes(w.getnframes())
>     cut = frames[:w.getframerate() * 5 * w.getnchannels() * w.getsampwidth()]  # 前5秒
> with wave.open('reminder.wav', 'wb') as w:
>     w.setparams(params)
>     w.writeframes(cut)
> ```

### 天气查询

1. 确保 `data/config.yaml` 中天气配置已启用：
   ```yaml
   weather:
     enabled: true
     city: "北京"
   ```
2. 右键托盘图标 → **"查看天气"**
3. 宠物旁的对话气泡显示城市、温度和湿度
4. 结果气泡 8 秒后自动消失
5. 在缓存刷新间隔内重复查询同一城市不会重复请求 API

> 默认使用 Open-Meteo 免 Key API，无需申请密钥。如需更高精度可切换到和风天气或 OpenWeatherMap。

### 网络时间校准

当发现提醒时间不准确时：

1. 右键托盘图标 → **"校准网络时间"**
2. 程序会连接阿里云 NTP 服务器获取标准时间
3. 自动计算本地时钟偏差并校正提醒引擎
4. 对话气泡显示校准结果，8 秒后自动消失

### 开机自启

1. 右键托盘图标 → **"开机自启"**
2. 通过 Windows 注册表添加/删除启动项
3. 带空格的程序路径会被正确引用；开发模式使用 `pythonw.exe` 启动

## 打包为 exe

### 一键打包

```powershell
python -m pip install -r requirements-build.txt
python build.py
```

默认生成无控制台窗口的正式版本，输出为 `dist/DesktopPet.exe`。需要排查启动问题时可构建控制台版本：

```powershell
python build.py --console
```

构建脚本会先校验入口、配置、素材、图标和 PyInstaller。构建过程使用 `build/` 临时目录，
只有新 EXE 完整生成后才整体替换 `dist/`，构建失败不会提前删除旧产物。成功发布的
`dist/` 只包含新 EXE，不会携带旧的 `data/`、日志或待处理提醒；需要保留的运行数据请在
打包前从 `dist/data/` 备份到其他目录。

### 打包后运行数据

```
dist/
└── DesktopPet.exe          # 单文件主程序

data/                       # 首次运行时在 exe 同级创建
├── config.yaml             # 用户配置
├── config.yaml.bak         # 最近一次成功保存的备份
└── logs/
    └── pet.log             # 自动轮转日志
```

> ⚠️ 打包后配置文件在 exe 同级 `data/` 目录，不在源码目录。

## 常见问题

### Q: 宠物窗口不显示？

A: 检查以下几点：
1. 确认程序已成功启动（托盘有图标）
2. 宠物可能被拖到屏幕外，右键托盘 → 切换活跃/安静模式让宠物回到默认位置
3. 查看 `data/logs/pet.log` 日志文件排查错误

### Q: 提醒没有触发？

A: 可能原因：
1. 提醒未启用 — 打开管理面板检查启用状态
2. 时间格式错误 — 确保是 `HH:MM` 格式（如 `18:30`）
3. 日期规则不匹配 — 检查“每天/工作日/休息日/指定日期”的选择
4. 程序未运行 — 确保托盘图标存在
5. 提醒发生在锁屏期间 — 解锁后会按 FIFO 队列展示

### Q: 为什么重复启动后没有第二个宠物？

A: 应用只允许一个实例运行。再次启动会激活已有宠物窗口，然后新进程立即退出。

### Q: 天气查询失败？

A: 可能原因：
1. 网络连接问题 — 检查是否能访问外网
2. 城市名错误 — 使用中文城市名（如"北京"、"上海"）
3. API 限制 — Open-Meteo 免费但有请求频率限制，建议不要频繁查询

### Q: 音效不播放？

A: 检查：
1. 系统音量是否静音
2. WAV 文件是否在 `assets/sounds/` 目录
3. 文件格式是否为 WAV（不支持 MP3，需转换）
4. 提醒配置中是否勾选了"播放提示音"

### Q: 如何更换宠物图片？

A: 替换 `assets/animations/` 下的 PNG 序列帧图片：
- `walk/` — 走动动画
- `cheer/` — 欢呼动画

也可以新增自定义子目录，提醒管理面板会自动扫描。

每帧按数字顺序命名（如 `frame_0.png`, `frame_1.png`...），保持相同尺寸。

### Q: 开机自启不生效？

A: 检查：
1. 杀毒软件可能拦截了注册表修改
2. 手动检查 `shell:startup` 目录是否有快捷方式
3. 以管理员身份运行一次程序再开启自启

### Q: 打包后体积太大？

A: 优化建议：
1. 减少动画帧数量或压缩 PNG 图片
2. 音效文件使用较低的采样率（如 22kHz 单声道）
3. 使用 `--exclude-module` 排除不需要的 PyQt5 模块
