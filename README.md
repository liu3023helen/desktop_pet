## 快速开始

### 环境要求

- Windows 10/11
- Python 3.8+
- 网络连接（天气查询、NTP 校时需要）

### 1. 安装依赖

```powershell
cd desktop_pet
python -m pip install -r requirements.txt
```

如遇网络问题，使用清华镜像：

```powershell
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

### 2. 运行程序

```powershell
python main.py
```

程序启动后：
1. 系统托盘出现宠物图标 🐾
2. 宠物出现在桌面右下角（默认安静模式）
3. 右键托盘图标打开操作菜单

### 3. 首次配置

首次运行时会自动将 `config.yaml` 复制到 `data/config.yaml`（exe 同级目录）。

推荐先通过 **"管理提醒"** 面板配置你的打卡时间和提醒文案，然后保存即可。

## 使用指南

### 启动与退出

| 操作 | 说明 |
|------|------|
| 双击 `main.py` 或 `DesktopPet.exe` | 启动程序 |
| 右键托盘图标 → **退出** | 完全退出程序 |
| 关闭宠物窗口 | 最小化到托盘（不退出） |

### 托盘菜单操作

右键点击系统托盘的宠物图标，弹出菜单：

| 菜单项 | 功能 |
|--------|------|
| 🐾 **活跃模式 / 安静模式** | 切换宠物行为状态 |
| ⏰ **管理提醒** | 打开提醒任务管理面板 |
| 🌤️ **查看天气** | 查询当前城市天气，结果以气泡+托盘通知展示 |
| ⏱️ **校准网络时间** | 从阿里云 NTP 获取标准时间并校正 |
| 🩺 **运行自检** | 检查配置、素材、NTP 和天气 API 连通性 |
| 🔌 **开机自启** | 开启/关闭开机自动启动 |
| ❌ **退出** | 完全退出程序 |

### 宠物交互

| 操作 | 效果 |
|------|------|
| **拖拽宠物** | 按住左键拖动，可移动到屏幕任意位置 |
| **右键宠物** | 弹出快捷菜单（活跃/安静模式切换、退出） |
| **闹钟触发** | 根据动作类型显示通知、播放动画或打开 URL，并显示操作窗口 |
| **安静模式** | 宠物停在右下角，显示静态图片，不闲逛 |
| **活跃模式** | 宠物在屏幕中央区域左右移动，播放循环动画 |

### 管理提醒任务

通过托盘菜单 → **"管理提醒"** 打开管理面板：

#### 新增提醒

1. 点击 **"新增提醒"** 按钮
2. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 提醒名称 | 任务的简短描述 | `下班打卡` |
| 触发时间 | 24小时制 HH:MM | `18:30` |
| 仅工作日 | 勾选后仅在周一至周五触发 | ☑ |
| 动作类型 | 触发时的行为 | `打开URL` / `播放动画` / `仅通知` |
| 目标URL | 动作类型为"打开URL"时填写 | `dingtalk://...` |
| 专属动画 | 触发时播放的动画 | `cheer` / `walk` |
| 提醒文案 | 气泡中显示的文本 | `下班啦！快去打卡~` |
| 播放提示音 | 是否播放音效 | ☑ |
| **专属音效** | 选择此提醒专用的 WAV 文件（留空则用默认提示音） | `小新-下班了.wav` |

3. 点击 **"确定"** 保存

#### 编辑/删除提醒

- 在列表中选中一行，点击 **"编辑"** 修改配置
- 点击 **"删除"** 移除该提醒
- 勾选/取消勾选启用列，快速开关提醒

#### 刷新列表

修改保存后自动刷新，无需重启程序。

#### 贪睡、跳过与完成

提醒触发时会显示非阻塞操作窗口：

- **稍后 5/10 分钟**：到期后再次触发该提醒
- **今天跳过**：当天不再触发同名提醒
- **已完成**：结束本次提醒周期

这些状态只在当前运行周期内保存，跨天会自动清理，程序重启后重置。

### 配置说明

配置文件位于 `data/config.yaml`（exe 同级目录），支持以下配置项：

#### 宠物基础配置

```yaml
pet:
  name: "小新"              # 宠物名称
  style: "shinchan"         # 风格标识
  default_animation: "cheer" # 默认动画
```

#### 提醒配置

```yaml
reminders:
  - name: "下班打卡"
    enabled: true             # 是否启用
    time: "18:30"            # 触发时间（24小时制）
    weekdays_only: false      # 仅工作日触发
    action_type: "open_url"   # open_url / play_animation / notify_only
    action_target: "dingtalk://..."  # 目标URL
    animation: "cheer"        # 专属动画
    message: "下班啦！快去打卡~"     # 提醒文案
    sound: true               # 是否播放提示音
    sound_file: ""            # 专属音效文件路径（空=使用默认）
```

#### 网络时间校准

```yaml
time_sync:
  enabled: true
  ntp_server: "ntp.aliyun.com"   # NTP 服务器
  tolerance_seconds: 30          # 偏差告警阈值（秒）
```

#### 天气配置

```yaml
weather:
  enabled: true
  city: "北京"                    # 查询城市
  api_provider: "openmeteo"       # openmeteo(免Key) / qweather / openweathermap
  api_key: ""                     # 和风天气/OpenWeatherMap 需填入 Key
  update_interval_minutes: 60     # 同一城市缓存刷新间隔（分钟）
```

### 自定义音效

#### 支持的格式

- **WAV 格式**（推荐）— 使用 `winsound` 播放，零额外依赖，无窗口闪烁
- 文件放在 `assets/sounds/` 目录下

#### 设置默认提示音

将默认提示音文件命名为 `reminder.wav`，放在 `assets/sounds/` 目录。

#### 为特定提醒设置专属音效

1. 将 WAV 文件放入 `assets/sounds/` 目录
2. 在提醒管理面板中，选择该提醒的 **"专属音效"** 下拉框
3. 选择对应的 WAV 文件
4. 保存即可

> 💡 **小技巧**：可以用 Python 脚本裁剪现有 WAV 文件的前几秒作为提示音：
> ```python
> import wave, os
> with wave.open('source.wav', 'rb') as w:
>     params = w.getparams()
>     frames = w.readframes(w.getnframes())
>     cut = frames[:w.getframerate() * 5 * w.getnchannels() * w.getsampwidth()]  # 前5秒
> with wave.open('reminder.wav', 'wb') as w:
>     w.setparams(params)
>     w.writeframes(cut)
> ```

### 天气查询

1. 确保 `data/config.yaml` 中天气配置已启用：
   ```yaml
   weather:
     enabled: true
     city: "北京"
   ```
2. 右键托盘图标 → **"查看天气"**
3. 天气信息会以两种方式展示：
   - **漫画气泡**：宠物旁弹出白色圆角气泡，显示城市、温度、湿度
   - **托盘通知**：系统托盘弹出通知消息
4. 气泡约 6 秒后自动消失
5. 在缓存刷新间隔内重复查询同一城市不会重复请求 API

> 默认使用 Open-Meteo 免 Key API，无需申请密钥。如需更高精度可切换到和风天气或 OpenWeatherMap。

### 网络时间校准

当发现提醒时间不准确时：

1. 右键托盘图标 → **"校准网络时间"**
2. 程序会连接阿里云 NTP 服务器获取标准时间
3. 自动计算本地时钟偏差并校正提醒引擎
4. 托盘通知显示校准结果

### 开机自启

1. 右键托盘图标 → **"开机自启"**
2. 通过 Windows 注册表添加/删除启动项
3. 带空格的程序路径会被正确引用；开发模式使用 `pythonw.exe` 启动

## 常见问题

### Q: 宠物窗口不显示？

A: 检查以下几点：
1. 确认程序已成功启动（托盘有图标）
2. 宠物可能被拖到屏幕外，右键托盘 → 切换活跃/安静模式让宠物回到默认位置
3. 查看 `data/logs/pet.log` 日志文件排查错误

### Q: 提醒没有触发？

A: 可能原因：
1. 提醒未启用 — 打开管理面板检查启用状态
2. 时间格式错误 — 确保是 `HH:MM` 格式（如 `18:30`）
3. 工作日限定 — 如果在周末，且勾选了"仅工作日"，不会触发
4. 程序未运行 — 确保托盘图标存在

### Q: 天气查询失败？

A: 可能原因：
1. 网络连接问题 — 检查是否能访问外网
2. 城市名错误 — 使用中文城市名（如"北京"、"上海"）
3. API 限制 — Open-Meteo 免费但有请求频率限制，建议不要频繁查询

### Q: 音效不播放？

A: 检查：
1. 系统音量是否静音
2. WAV 文件是否在 `assets/sounds/` 目录
3. 文件格式是否为 WAV（不支持 MP3，需转换）
4. 提醒配置中是否勾选了"播放提示音"

### Q: 如何更换宠物图片？

A: 替换 `assets/animations/` 下的 PNG 序列帧图片：
- `walk/` — 走动动画
- `cheer/` — 欢呼动画

也可以新增自定义子目录，提醒管理面板会自动扫描。

每帧按数字顺序命名（如 `frame_0.png`, `frame_1.png`...），保持相同尺寸。

### Q: 开机自启不生效？

A: 检查：
1. 杀毒软件可能拦截了注册表修改
2. 手动检查 `shell:startup` 目录是否有快捷方式
3. 以管理员身份运行一次程序再开启自启

### Q: 打包后体积太大？

A: 优化建议：
1. 减少动画帧数量或压缩 PNG 图片
2. 音效文件使用较低的采样率（如 22kHz 单声道）
3. 使用 `--exclude-module` 排除不需要的 PyQt5 模块
