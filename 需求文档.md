# Windows11 桌面打卡电子宠物 需求文档（v4 - 开发中）

## 一、项目总目标

基于联想小新Win11笔记本，开发轻量化本地桌面电子宠物程序；**一期实现本地独立运行**，通过配置文件驱动可扩展提醒引擎，首期仅接入"下班打卡"场景；预留扩展能力，后期新增提醒（如运动、喝水等）只需修改配置无需改代码。**二期增加GUI管理面板和网络能力**，降低使用门槛并提升准确性。

## 三、分模块验收标准（可量化、可自测）

### 模块1：基础桌面宠物窗口
1. ✅ 运行后生成无边框、透明置顶窗口，任何软件窗口不会遮挡宠物；
2. ✅ 按住宠物可拖拽至桌面任意位置；
3. ✅ 播放cheer动画，帧率稳定，闲置CPU占用≤5%，内存占用≤100MB；
4. ✅ 右键弹出菜单：隐藏至托盘、退出程序；托盘图标常驻任务栏右下角；
5. ✅ 适配高分屏（DPI缩放已启用）；
6. ✅ 多显示器场景下宠物不会跑到副屏消失。

### 模块2：可扩展提醒引擎（配置驱动）
1. ✅ 启动时读取 `config.yaml` 中的 `reminders` 列表，解析每条提醒的触发时间、动作类型、动画、文案；
2. ✅ 内置定时器精度误差≤5秒，系统休眠唤醒后能自动校正时间继续工作；
3. ✅ 提醒触发时：宠物切换对应专属动画 → 弹出Windows系统通知（显示自定义文案）→ 执行指定动作；
4. ✅ 动画持续10秒后自动恢复默认状态；
5. ✅ 支持静音开关（音效播放已实现）；
6. ✅ **扩展性验证**：在配置文件中新增一条提醒规则后重启程序，能正常触发且无需改代码。

### 模块3：钉钉打卡跳转能力
1. ✅ 本地存在钉钉客户端：通过协议 `dingtalk://` 自动启动钉钉并跳转工作台打卡页面；
2. ✅ 未安装钉钉：自动调用默认浏览器打开钉钉网页版打卡地址；
3. ✅ 跳转失败写入日志，不导致程序崩溃；
4. ✅ 支持配置自定义打卡URL（兼容未来可能的办公平台切换）；
5. ✅ URL白名单校验，拒绝命令注入特征。

### 模块4：打包与部署适配
1. ✅ 打包输出单文件exe，体积≤70MB，无需安装Python环境，双击直接运行；
2. ✅ 支持添加至注册表启动项，开机后台静默启动（无额外窗口弹出）；
3. ✅ 程序休眠、唤醒后可正常运行；
4. ✅ 完整素材（图片、默认配置）内置打包，首次运行时释放 `config.yaml` 到数据目录；
5. ❌ 提供卸载指引（清理启动项注册表、用户目录配置文件）。

### 模块5：闹钟任务管理面板（二期）
1. ❌ 托盘菜单增加"管理提醒"入口，点击弹出独立设置窗口；
2. ❌ 设置窗口以表格形式列出所有提醒任务（名称、时间、是否启用、操作）；
3. ❌ 支持新增任务：弹窗表单填写名称、时间、动作类型、文案等字段；
4. ❌ 支持编辑已有任务：点击编辑按钮回填表单，保存后覆盖原配置；
5. ❌ 支持删除任务：点击删除按钮，二次确认后移除；
6. ❌ 支持启用/禁用切换：每行提供复选框或开关按钮；
7. ❌ 修改保存后实时生效，无需重启程序（通知提醒引擎重新加载配置）；
8. ❌ 数据持久化到 `data/config.yaml`，格式与现有配置兼容。

### 模块6：网络时间校准（二期）
1. ❌ 启动时自动从NTP服务器获取网络时间（如 `ntp.aliyun.com` 或 `time.windows.com`）；
2. ❌ 校准时与本地时间对比，偏差超过阈值（如30秒）时在托盘区域弹出提示；
3. ❌ 不自动修改系统时间（需要管理员权限），仅提示用户"本地时间可能不准确"；
4. ❌ 提醒引擎内部使用网络时间作为基准，而非本地系统时间，确保触发准确；
5. ❌ 可配置校准频率（默认每天一次），失败时不阻塞程序启动；
6. ❌ 设置面板提供"立即校准"按钮和校准结果展示。

### 模块7：天气信息（二期）
1. ✅ 启动时调用 Open-Meteo API 获取指定城市天气信息（温度、天气状况如"晴/雨/阴"、湿度）；
2. ✅ 使用 Open-Meteo 免 Key API，无需用户申请；
3. ✅ 天气信息通过漫画对话气泡在宠物旁展示，6秒后自动消失；
4. ✅ 可配置城市（默认北京）、更新频率（默认每小时）；
5. ✅ 网络请求失败时静默降级，不影响其他功能；
6. ✅ 托盘菜单"查看天气"触发异步查询，结果以气泡+托盘通知双通道展示。

### 模块7.5：漫画对话气泡（二期）
1. ✅ 自定义 `BubbleWidget` 控件，使用样式表实现漫画风格气泡（白色背景+黑色粗边框+圆角）；
2. ✅ 气泡为独立置顶窗口，自动定位在宠物上方（空间不足时在下方），不超出屏幕边界；
3. ✅ 闹钟触发时气泡显示提醒文案（如"测试~~"），8秒后自动消失；
4. ✅ 天气查询时气泡显示天气信息（如"北京 · 多云 · 27°C\n湿度 67%"），6秒后自动消失；
5. ✅ 自动调整尺寸适应文字内容，最大宽度300px自动换行；
6. ✅ 纯代码实现，无额外素材依赖，不引入第三方库。

### 模块8：提醒交互增强（二期）
1. ❌ 提醒触发时弹窗增加"贪睡"按钮（延迟5/10分钟再次提醒）；
2. ❌ 提醒触发时弹窗增加"今天跳过"按钮（今日不再触发此提醒）；
3. ❌ 提醒触发时弹窗增加"已完成"按钮（立即关闭提醒并记录）；
4. ❌ 贪睡/跳过状态持久化到配置或临时文件，程序重启后失效。

### 模块9：工作日/节假日智能识别（二期）
1. ❌ 提醒任务支持设置"仅工作日触发"选项；
2. ❌ 内置中国法定节假日调休规则（可通过配置文件或网络接口更新）；
3. ❌ 周末和法定节假日自动跳过标记为"工作日"的提醒；
4. ❌ 调休上班日即使为周六日也正常触发提醒。

## 附录A：EXE体积影响分析

### 当前打包体积构成（约61MB）

| 组成部分 | 体积估算 | 说明 |
|----------|---------|------|
| Python解释器运行时 | ~15MB | PyInstaller bootstrap + Python3.x core |
| PyQt5 框架 | ~35MB | Qt核心+GUI模块+平台插件（最大头） |
| Pillow + numpy | ~6MB | 图像处理依赖 |
| 项目代码(.py) | <0.5MB | 所有Python源码合计 |
| 动画素材(assets/) | ~3MB | PNG序列帧+图标 |
| PyYAML + schedule | ~1MB | 轻量级纯Python库 |
| UPX压缩后节省 | -5~8MB | 可执行文件压缩 |

### 漫画气泡功能对体积的影响

**几乎为零**，原因如下：

| 项目 | 增量 | 说明 |
|------|------|------|
| `bubble_widget.py` 源码 | ~3KB | 约100行纯Python代码 |
| 新导入模块 | 0 | 仅使用 `PyQt5.QtWidgets` / `PyQt5.QtCore` / `PyQt5.QtGui`，已在项目中被大量引用 |
| 新依赖库 | 0 | 不引入任何第三方包 |
| 图片素材 | 0 | 气泡完全用 CSS 样式表绘制（border-radius + border伪元素画三角箭头） |
| **合计增量** | **<0.1MB** | 打包后几乎不可感知 |
