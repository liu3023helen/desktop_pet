# Windows11 桌面打卡电子宠物 需求文档（v3 - 开发中）

## 一、项目总目标

基于联想小新Win11笔记本，开发轻量化本地桌面电子宠物程序；**一期实现本地独立运行**，通过配置文件驱动可扩展提醒引擎，首期仅接入"下班打卡"场景；预留扩展能力，后期新增提醒（如运动、喝水等）只需修改配置无需改代码。

## 三、分模块验收标准（可量化、可自测）

### 模块1：基础桌面宠物窗口
1. ✅ 运行后生成无边框、透明置顶窗口，任何软件窗口不会遮挡宠物；
2. ✅ 按住宠物可拖拽至桌面任意位置；
3. ✅ 播放cheer动画，帧率稳定，闲置CPU占用≤5%，内存占用≤100MB；
4. ✅ 右键弹出菜单：隐藏至托盘、退出程序；托盘图标常驻任务栏右下角；
5. ⚠️ 适配小新笔记本高分屏（DPI缩放已启用，需实测验证无拉伸无黑边）；
6. ❌ 多显示器场景下宠物不会跑到副屏消失（当前未做主屏限制）。

### 模块2：可扩展提醒引擎（配置驱动）
1. ✅ 启动时读取 `config.yaml` 中的 `reminders` 列表，解析每条提醒的触发时间、动作类型、动画、文案；
2. ✅ 内置定时器精度误差≤5秒，系统休眠唤醒后能自动校正时间继续工作；
3. ✅ 提醒触发时：宠物切换对应专属动画 → 弹出Windows系统通知（显示自定义文案）→ 执行指定动作；
4. ✅ 动画持续10秒后自动恢复默认状态；
5. ❌ 支持静音开关（配置项已预留，音效播放未实现）；
6. ✅ **扩展性验证**：在配置文件中新增一条提醒规则后重启程序，能正常触发且无需改代码。

### 模块3：钉钉打卡跳转能力
1. ✅ 本地存在钉钉客户端：通过协议 `dingtalk://` 自动启动钉钉并跳转工作台打卡页面；
2. ✅ 未安装钉钉：自动调用默认浏览器打开钉钉网页版打卡地址；
3. ✅ 跳转失败写入日志，不导致程序崩溃；
4. ✅ 支持配置自定义打卡URL（兼容未来可能的办公平台切换）；
5. ✅ URL白名单校验，拒绝命令注入特征。

### 模块4：打包与部署适配
1. ❌ 打包输出单文件exe，体积≤50MB，无需安装Python环境，双击直接运行；
2. ❌ 支持添加至系统启动文件夹，开机后台静默启动（无额外窗口弹出）；
3. ⚠️ 程序休眠、唤醒后可正常运行（PyQt5理论上支持，需实测验证）；
4. ✅ 完整素材（图片、默认配置）内置打包，首次运行时释放 `config.yaml` 到用户目录；
5. ❌ 提供卸载指引（清理启动项注册表、用户目录配置文件）。

## 四、提醒配置格式规范

```yaml
# config.yaml - 电子宠物提醒配置
# 首次启动自动生成，路径：%APPDATA%\DesktopPet\config.yaml

pet:
  name: "小新"
  style: "shinchan"        # 角色风格标识
  default_animation: "cheer" # 默认动画

reminders:
  - name: "下班打卡"
    enabled: true
    time: "18:30"           # 24小时制 HH:MM
    action_type: "open_url" # 动作类型: open_url / play_animation / notify_only
    action_target: "dingtalk://dingtalkclient/page/link?pc_slide=false&url=https://im.dingtalk.com/attendancemobile/index.html"
    animation: "cheer"      # 专属动画: cheer/run/stretch/custom
    message: "下班啦！快去打卡~"
    sound: true             # 是否播放音效
    
  # --- 以下为扩展示例（默认注释）---
  # - name: "运动提醒"
  #   enabled: false
  #   time: "20:00"
  #   action_type: "play_animation"
  #   animation: "run"
  #   message: "该起来动一动了！"
  #   sound: true
```

## 五、电子宠物角色与素材

### 角色
直接使用蜡笔小新图片作为桌面宠物角色。

### 素材来源
- 源文件：`D:\helen\code_test\蜡笔小新-310px_爱给网_欢呼.png`（310x310 RGBA 透明背景）

### 动画方案
极简2帧交替循环：
| 帧 | 来源 | 说明 |
|----|------|------|
| frame_000.png | 原图缩放至256x256 | 抬右脚姿态 |
| frame_001.png | frame_000 水平翻转 | 抬左脚姿态 |

两帧以 ~3fps 频率循环播放，形成简单的原地踏步/欢呼效果。

### 素材技术要求
- PNG带透明alpha通道
- 输出尺寸 256x256
- 窗口大小与素材一致，无边框透明置顶显示
